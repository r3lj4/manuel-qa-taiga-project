# Manual QA Testing – Taiga Project

## Project Overview
This repository contains a **manual QA testing portfolio project** created for learning and professional development purposes.  
The goal of this project is to demonstrate practical knowledge of **manual software testing standards, techniques, and documentation** using a real, open-source web application.

The project covers the full manual QA process, including test planning, test case design, defect reporting, regression testing, and exploratory testing.

---

## Application Under Test
**Taiga** – Open-source project management and agile planning tool.

- Application type: Web application
- URL: https://tree.taiga.io
- Domain: Project & task management (Agile/Scrum)
- Authentication: Required (registered users)

Taiga was selected because it is a real-world, open-source application with complex workflows similar to tools like Jira or Trello.

---

## Scope of Testing
The following functionalities are included in the testing scope:

- User registration
- Login and logout
- Project creation
- User stories and task management
- Task status workflow
- Input field validations
- Basic permission handling
- UI behavior and error handling

Out of scope:
- Performance testing
- Automation testing
- Backend and database testing

---

## Types of Testing Performed
- Functional testing
- Smoke testing
- Regression testing
- Exploratory testing
- UI validation
- Negative testing

Black-box testing techniques applied:
- Equivalence Partitioning
- Boundary Value Analysis
- Error Guessing

---

## Test Artifacts
This repository includes the following QA documentation:

- **Test Plan** – testing approach, scope, risks, and environment
- **Test Scenarios** – high-level testing scenarios
- **Test Cases** – detailed manual test cases with expected results
- **Bug Reports** – documented defects with severity and priority
- **Regression Checklist** – core functionality verification
- **Exploratory Testing Notes** – session-based findings

Each artifact is organized in a separate folder for clarity.
